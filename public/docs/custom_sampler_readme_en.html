How to Use "Create Custom Sampler" Block

This block allows you to create a completely new instrument from your own audio samples.

How it works: Resampling

The core of this block is based on Tone.js's powerful Sampler, which relies on Digital Audio Resampling:

  • Single Sample: If you provide only one sample (e.g., a C4 violin recording), the sampler can generate the full range of notes by speeding up or slowing down playback.
  • Multisampling: For higher quality, you can provide multiple samples at different pitches. The sampler will automatically pick the closest recording to ensure the best sound quality.

Step 1: Prepare Your Samples

  1. Find Samples: Search for single-note audio files (.wav, .mp3). Sites like Freesound.org are great resources.
  2. Check License: Ensure the files are free to use (e.g., CC0 or Public Domain).

Step 2: Host Your Audio Online

To use your samples, they must be accessible via a public URL. GitHub Pages is a highly recommended free method:

  1. Create a new public GitHub repository (e.g., my-samples).
  2. Upload your files and commit the changes.
  3. Enable GitHub Pages in Settings > Pages.
  4. Get your base URL, e.g., https://user.github.io/my-samples/

Step 3: Configure the Block

Drag the "Instrument Container" from the Create Instruments category, give it a name, and then place the "Create Sampler Type..." block inside it:

  • Container Name: Give your instrument a name (e.g., MyViolin).
  • Sampler Type: Select from the list or choose "Custom URL...".
  • Base URL: Paste your GitHub Pages URL (if custom).
  • Sample Map: Use a JSON string to map note names to filenames (if custom).

Example Format: '{"A4": "violin-a4.wav", "C5": "violin-c5.wav"}'

Note: The entire string must be wrapped in single quotes, while keys and values inside must use double quotes.

Supported Formats

We recommend .wav (best quality) or .mp3 (best compatibility). Modern browsers also support .m4a, .ogg, and .flac.

Sampler and ADSR Envelope

Unlike synthesizers, Samplers play pre-recorded files. The impact of ADSR parameters is as follows:

  • Release - Most Important: Controls how fast the sound fades after a key is released. High values add resonance; low values cut the sound abruptly.
  • Attack: Controls the fade-in speed. High values can transform a sharp piano into a soft ambient pad.
  • Decay & Sustain: Since samples have built-in natural dynamics, these two have minimal effect on realism and are typically left at default values.
