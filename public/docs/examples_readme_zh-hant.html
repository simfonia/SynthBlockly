範例專案使用說明

透過範例專案，您可以快速了解如何使用 SynthBlockly 建立各種音樂效果與互動裝置。

1. 如何載入範例

  • 點擊工具列上的「範例專案」圖示（圖示呈現資料夾或🎹形狀）。
  • 在彈出的清單中選擇您感興趣的專案。
  • 確認載入後，現有的工作區內容將會被範例取代。

2. 範例類型

🎹 僅限 Blockly (Software)： 這些範例不需要外部硬體，直接點擊「執行積木」即可聽到聲音。

🤖 硬體整合 (Hardware)： 這些範例需要配合 Arduino 或其他實體電子元件。載入後會自動彈出硬體指引。

3. 硬體範例的操作流程

  1. 載入範例後，請詳閱彈出的「硬體設定指引」。
  2. 點擊「下載 Arduino 程式碼 (.ino)」按鈕，取得對應的 C++ 程式。
  3. 使用 Arduino IDE 開啟下載的檔案，並燒錄至您的開發板。
  4. 根據說明完成電路接線（例如按鈕、可變電阻或感測器）。
  5. 回到 SynthBlockly，點擊工具列的「連接序列埠」與您的 Arduino 連線。

提示： 您可以修改範例的積木參數來實驗不同的聲音效果！

4. 如何擴充範例 (給開發者)

如果您想新增自己的範例到清單中，請依循以下規範將檔案放置於專案的 src/examples/ 目錄，系統會自動根據副檔名進行掃描：

  • 純軟體範例： 直接在根目錄放入 .xml 檔案 (例如：src/examples/MySong.xml)。
  • 硬體範例 (資料夾型)： 建立一個資料夾 (例如：src/examples/MyProject/)，系統會自動匹配其中的檔案：
    • Blockly 程式 (入口)： 檔名必須是 main.xml 或是與資料夾同名 (如 MyProject.xml)。
    • Arduino 程式： 只要是 .ino 檔案即可，檔名不限，且可以放在該資料夾下的任何位置。
    • 說明文件： 只要是 .md 檔案即可，檔名不限 (如 README.md)，系統會將其內容顯示在硬體指引視窗中。
